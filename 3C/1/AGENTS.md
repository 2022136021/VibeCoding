# 🛑 자율성 및 제약 사항 (Autonomy & Boundaries)
- **자율 작업 범위:** 에이전트는 각 마일스톤(Step) 내에서 코드 작성, 린트(Lint) 검사, 단위 테스트 실행을 자율적으로 수행한다.
- **승인 필수 작업:** 하나의 Step을 완료하고 다음 Step으로 넘어가기 전, 반드시 작업 내역을 요약 보고하고 사용자의 진행 승인(안티그래피티 진행 허가)을 받아야 한다.

# 🛠️ Agent Skills (사용 가능한 스킬 및 도구)
에이전트는 작업을 수행할 때 아래의 스킬을 적극 활용해야 한다.
1. **코드 품질 검증:** 코드 수정 후 터미널에서 문법 및 에러 체크 명령어를 실행하여 스스로 디버깅한다.
2. **브라우저 시각화 테스트:** UI/UX 수정 시, 안티그래비티의 내장 브라우저 자동화 기능을 통해 화면이 깨지지 않는지 시각적으로 확인한다.

# 🔄 작업 프로세스 (5단계 마일스톤)
에이전트는 아래 5단계에 따라 순차적으로 작업을 진행해야 한다. **각 단계를 완료할 때마다 'Agent Skills'가 에러 없이 잘 동작하는지 본인이 반드시 직접 확인(Verify)한 후**, 다음 단계로 진행할지 사용자에게 묻는다.

- **Step 1: 기본 구조 및 핵심 기능 구현**
  - 프로젝트의 기본 뼈대를 잡고 가장 필수적인 핵심 로직을 작성한다.
  - 작업 완료 후, 코드가 정상적으로 실행되는지 기본 테스트를 거친다.

- **Step 2: 카테고리 기능 및 UI 개선**
  - 핵심 기능 위에 카테고리 분류를 추가하고 전반적인 UI를 다듬는다.
  - 브라우저 스킬을 활용해 추가된 UI가 의도대로 렌더링되는지 확인한다.

- **Step 3: 진행률 대시보드 추가**
  - 사용자가 작업 상태를 파악할 수 있는 대시보드 컴포넌트를 구현한다.
  - 실제 데이터(혹은 모킹 데이터)가 화면에 올바르게 연동되어 표시되는지 검증한다.

- **Step 4: 다크 모드 및 고급 기능**
  - 다크 모드 토글 등 사용자 편의를 위한 심화 기능을 적용한다.
  - 모드 전환 시 상태(State)가 유지되고 전역 스타일이 정상적으로 변경되는지 테스트한다.

- **Step 5: 최종 완성 및 최적화**
  - 전체 코드를 리팩토링하고, 불필요한 코드를 제거하여 최적화한다.
  - 모든 기능이 통합된 상태에서 최종 빌드 및 전체 테스트를 통과하는지 확인 후 최종 보고한다.
